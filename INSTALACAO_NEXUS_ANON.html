Instalação local assistida

NEXUS ANON

Guia simples para instalar e abrir o NEXUS ANON em computador Windows, com execução local pelo navegador, sem envio de documentos para a internet durante o processamento.

Tutorial simplificado de instalação NEXUS ANON

1

Instale os programas obrigatórios

Antes do NEXUS ANON funcionar, o computador precisa ter Python 3.11 ou 3.12, Node.js LTS e Ollama instalados. Eles são a base local que permite ao sistema abrir a tela, processar documentos e acionar obrigatoriamente o modelo de IA instalado na máquina.

2

Execute o instalador do NEXUS ANON

Dê dois cliques em INSTALAR_NEXUS_ANON.bat. Esse arquivo prepara o backend, a interface, as dependências e confere o modelo local padrão qwen3:32b, usado diretamente pelo ANON.

3

Abra o programa

Dê dois cliques em ABRIR_NEXUS_ANON.bat ou no atalho NEXUS ANON criado na Área de Trabalho. O sistema liga os serviços locais, tenta iniciar o Ollama quando necessário e abre automaticamente o navegador em http://localhost:5173/.

O que precisa estar instalado

Item Finalidade
Python 3.11 ou 3.12 Executa o motor local de processamento, leitura de documentos, anonimização, exportação e cálculo de hash.
Node.js LTS Abre a interface web local usada pelo navegador.
Ollama Executa o modelo de IA local, sem envio de texto para serviços externos. Deve permanecer em execução durante a anonimização.
qwen3:32b Modelo local padrão usado diretamente pelo ANON. A especialização fica nos prompts, perfis, regras e validadores internos.
Tesseract OCR Opcional. Necessário somente para melhorar leitura de imagem ou PDF escaneado quando o OCR for utilizado.
LibreOffice Opcional. Recomendado para futura conversão de arquivos .DOC legados.

Arquivos principais do instalador

  • INSTALAR_NEXUS_ANON.bat: prepara o computador para rodar o sistema.
  • install.py: assistente alternativo em Python para instalar dependências do backend e da interface.
  • install_check.py: verifica rapidamente Python, Node.js, npm, Ollama, estrutura do ANON e modelos locais.
  • ABRIR_NEXUS_ANON.bat: liga o backend, tenta iniciar o Ollama, liga a interface e abre o navegador.
  • NEXUS_ANON.url: link local direto para a interface, útil quando os serviços já estão ligados.
  • GERAR_PACOTE_NEXUS_ANON.bat: cria um pacote ZIP limpo para transporte ou entrega institucional.

Importante: o arquivo qwen3:32b é muito grande e fica armazenado pelo Ollama no computador. Por isso, o pacote do NEXUS ANON não deve carregar esse modelo internamente. O instalador verifica se ele já existe e pode solicitar o download pelo próprio Ollama.

Ordem recomendada de instalação

  1. Instale Python 3.11 ou 3.12. Durante a instalação, marque a opção Add python.exe to PATH.
  2. Instale Node.js LTS. Após instalar, feche e abra novamente o instalador do NEXUS ANON.
  3. Instale Ollama e confirme que ele está funcionando no Windows. O Ollama precisa estar aberto ou responder em segundo plano para a anonimização ocorrer.
  4. Dê dois cliques em INSTALAR_NEXUS_ANON.bat.
  5. Se o instalador perguntar sobre o download do qwen3:32b, responda S se desejar baixar o modelo naquele momento.
  6. Ao final, use ABRIR_NEXUS_ANON.bat ou o atalho da Área de Trabalho.

Atenção: o NEXUS ANON processa documentos sensíveis localmente, mas o resultado deve ser sempre revisado por pessoa responsável antes de uso oficial, compartilhamento, impressão ou juntada a qualquer procedimento.

Requisitos de máquina

  • Mínimo prático: Windows 10/11, SSD, 32 GB de RAM e espaço livre suficiente para modelo e dependências.
  • Recomendado: 64 GB de RAM, SSD rápido e GPU compatível quando disponível.
  • Espaço em disco: reserve dezenas de GB para o modelo qwen3:32b, dependências e documentos processados.
  • Internet: necessária apenas para baixar programas, dependências e modelo na instalação inicial. O processamento documental deve ocorrer localmente.

Problemas comuns

  • Se abrir http://127.0.0.1:8000/ e aparecer "Not Found", isso ? normal: essa porta ? da API. A tela correta ? http://localhost:5173/.
  • Se o botão abrir e nada acontecer, execute novamente ABRIR_NEXUS_ANON.bat e aguarde alguns segundos.
  • Se o Ollama não for encontrado ou não estiver em execução, instale o Ollama, abra o Ollama manualmente e tente novamente. A interface pode abrir sem ele, mas a anonimização será bloqueada.
  • Se o modelo for lento, use máquina mais potente ou reduza o tamanho/lote dos documentos processados.

O que o instalador faz automaticamente

  • Cria o ambiente Python local do backend.
  • Instala as bibliotecas necessárias para FastAPI, leitura de PDF, DOCX e exportação.
  • Instala as dependências da interface React.
  • Verifica se o Ollama está disponível e tenta iniciar o serviço local quando possível.
  • Verifica se o qwen3:32b está instalado e oferece baixar quando estiver ausente.
  • Padroniza o uso do qwen3:32b, sem criar modelo derivado.
  • Cria um atalho na Área de Trabalho para abrir o sistema sem digitar comandos.
